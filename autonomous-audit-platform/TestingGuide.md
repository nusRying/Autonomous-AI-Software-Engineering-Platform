# Testing Guide: Autonomous Audit Platform (Milestone 1)

This guide provides three distinct testing scenarios to verify the capabilities of your new AI software auditor.

---

### **Scenario 1: The "Vulnerable" Project**
**Goal:** Verify static scanning and AI detection of security leaks.

1.  Create a folder named `vulnerable_test`.
2.  Add a file `main.py` with this content:
    ```python
    import os
    # CRITICAL: Hardcoded API Key
    STRIPE_SECRET = "sk_live_51P8zB2L..."
    
    def process_payment():
        # TODO: Fix insecure bypass here
        pass
    ```
3.  **Run Audit:** Enter the path to `vulnerable_test`.
4.  **Expect:** 
    *   **Health Score:** < 5
    *   **Finding 1:** "Hardcoded secret" (Static Scan)
    *   **Finding 2:** "Technical debt marker" (Static Scan)
    *   **Finding 3:** "AI CONFIRMED: High Risk Credential" (AI Synthesis)

---

### **Scenario 2: The "Broken Runtime" Project**
**Goal:** Verify Docker sandbox and runtime analysis.

1.  Create a folder named `runtime_test`.
2.  Add `main.py`:
    ```python
    import non_existent_library
    print("This will never run.")
    ```
3.  Add `requirements.txt`:
    ```text
    fastapi
    ```
4.  **Run Audit:** Enter the path to `runtime_test`.
5.  **Expect:**
    *   **Status:** Completed (Audit finishes even if code fails).
    *   **Findings:** "ModuleNotFoundError" in the Runtime Analysis section.
    *   **Logs:** The exact stack trace from the Docker container.

---

### **Scenario 3: The "Architecture" Project**
**Goal:** Verify documentation analysis and AI architectural mapping.

1.  Create a folder named `arch_test`.
2.  Add `README.md` with:
    ```markdown
    # Payment Microservice
    This project uses FastAPI and connects to a PostgreSQL database.
    Target users are developers looking for a secure billing engine.
    ```
3.  Add a basic `app.py`.
4.  **Run Audit:** Enter the path to `arch_test`.
5.  **Expect:**
    *   **Documentation Analysis:** A summary generated by LlamaIndex (or fallback) identifying the tech stack (FastAPI/Postgres).
    *   **AI Insight:** "Documentation Score" (1-10) based on README quality.
